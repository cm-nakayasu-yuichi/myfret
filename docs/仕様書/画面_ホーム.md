# 画面仕様書: ホーム

## 1. 概要

| 項目 | 値 |
| --- | --- |
| パス | `/` |
| 画面コンポーネント | `src/components/pages/home.tsx` (`HomePage`) |
| ヘッダ | あり |
| URL パラメータ | なし |
| 主目的 | 週間ランキング（曲・アーティスト）の表示とサイト内ナビゲーションの起点 |

## 2. レイアウト

`PageContainer` の中に `HomeContainer` を 1 つ配置する。`HomeContainer` は MUI `Grid` の 2 カラム構成。

```
┌──────────────────────────────────────────────────────────────┐
│ Header                                                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────────────────┐  ┌─────────────────────────┐  │
│  │  楽曲ランキング           │  │  アーティストランキング   │  │
│  │  [1] 曲名 - アーティスト  │  │  [1] アーティスト        │  │
│  │  [2] 曲名 - アーティスト  │  │  [2] アーティスト        │  │
│  │  ...                     │  │  ...                     │  │
│  │  もっと見る →            │  │  もっと見る →            │  │
│  └─────────────────────────┘  └─────────────────────────┘  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- `<Grid container spacing={4}>` で 2 つのセクションを並べる。
- ブレークポイント `md` 以上で 6:6 の横並び、`md` 未満で縦積み（各 `Grid item xs={12} md={6}`）。

## 3. データ取得

| 項目 | 値 |
| --- | --- |
| hook | `useGetHomeInfo` |
| API | `GET /api/home-info` |
| レスポンス型 | `HomeInfoResponse` |
| キャッシュ | `CacheKey.HOME_INFO`、TTL 1 週間 |

```ts
interface HomeInfoResponse {
    song_ranking: SongListItem[];
    artist_ranking: ArtistListItem[];
}
```

- ページマウント時に 1 回フェッチ。
- キャッシュヒット時は API を叩かず即座に表示。
- キャッシュ仕様の詳細は [キャッシュ仕様書](./キャッシュ仕様書.md) を参照。

## 4. セクション仕様

### 4.1 楽曲ランキングセクション

実装: `src/components/features/home/HomeSongRankingSection.tsx` (`HomeSongRankingSection`)

| 構成要素 | 内容 |
| --- | --- |
| 見出し | `SectionTitle`「楽曲ランキング」（件数表示なし） |
| 状態制御 | `ListContainer`（loading / empty / error / 通常） |
| リスト | `NormalList` |
| 各行 | `NormalListItem` に `RankingNumber` + `SongListText`、`to={'/song/${song.id}'}` |
| 末尾 | `MoreListItem`、`to="/ranking/songs"` |

- `state.result?.song_ranking` を順に並べる。
- `empty` 判定: `state.result?.song_ranking.length` が 0 以下のとき。
- 行のクリックで個別曲ページへ遷移。

### 4.2 アーティストランキングセクション

実装: `src/components/features/home/HomeArtistRankingSection.tsx` (`HomeArtistRankingSection`)

| 構成要素 | 内容 |
| --- | --- |
| 見出し | `SectionTitle`「アーティストランキング」（件数表示なし） |
| 状態制御 | `ListContainer` |
| リスト | `NormalList` |
| 各行 | `NormalListItem` に `RankingNumber` + `ArtistListText`、`to={'/artist/${artist.id}'}` |
| 末尾 | `MoreListItem`、`to="/ranking/artists"` |

- `state.result?.artist_ranking` を順に並べる。
- 行のクリックでアーティスト曲一覧ページへ遷移。

## 5. 状態と表示

| 状態 | 表示 |
| --- | --- |
| ローディング中 | 各セクションの `ListContainer` に `CircularProgress` |
| 取得エラー | 各セクションに「取得に失敗しました」 |
| 空配列 | 各セクションに「取得に失敗しました」 |
| 通常 | リスト + 「もっと見る」リンク |

両セクションは同じ `useGetHomeInfo` の戻り値（`loading` / `error` / `result`）を共有する。一方が成功してもう一方が空、というパターンは API 仕様上発生しない想定だが、`empty` 判定はセクションごとに行う。

## 6. ナビゲーション

| 操作 | 遷移先 |
| --- | --- |
| 曲ランキング行クリック | `/song/{songId}` |
| アーティストランキング行クリック | `/artist/{artistId}` |
| 「もっと見る」（曲） | `/ranking/songs` |
| 「もっと見る」（アーティスト） | `/ranking/artists` |
| ヘッダの検索・メニュー | [共通 UI・レイアウト仕様書 §3](./共通UI・レイアウト仕様書.md) を参照 |

「もっと見る」の遷移先と実際のルート定義はいずれも末尾スラッシュなし (`/ranking/songs`, `/ranking/artists`) で揃えている。

## 7. ランキング表示

順位の見た目は `RankingNumber`（[共通 UI・レイアウト仕様書 §5](./共通UI・レイアウト仕様書.md#5-rankingnumber)）に従う。

- 1 位: 金色、太字
- 2 位: 銀色、太字
- 3 位: 銅色、太字
- 4 位以降: 通常色、通常ウェイト
