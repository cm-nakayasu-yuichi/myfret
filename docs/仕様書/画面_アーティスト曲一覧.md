# 画面仕様書: アーティスト曲一覧

## 1. 概要

| 項目 | 値 |
| --- | --- |
| パス | `/artist/:name` / `/artist/:name/:page` |
| 画面コンポーネント | `src/components/pages/artist.tsx` (`ArtistPage`) |
| ルート定義 | `src/app/routes/artist.tsx` |
| ヘッダ | あり |
| URL パラメータ | `name`（アーティスト名）、`page`（ページ番号、省略可） |
| 主目的 | 指定アーティストの曲一覧をページネーション付きで表示 |

`name` は ufret.jp の `artist.php?data=` の値（実体はアーティスト名そのもの、もしくは ID 文字列）。曲ページのアーティストリンク・検索結果アーティストランキング・ホームのアーティストランキングからの遷移先。

## 2. データ取得

| 項目 | 値 |
| --- | --- |
| hook | `useGetSongsOfArtist(name)` |
| API | `GET /api/artist/:name` |
| レスポンス型 | `SongListResponse` |
| キャッシュ | なし |

```ts
interface SongListResponse {
    songs: SongListItem[];   // 各曲の id, name, artist, rank
    count: number;
}
```

- `name` パラメータをそのまま hook に渡す（hook 内部で `encodeURIWithPlus` がかかる）。
- `name` が変わるたびに再フェッチ。
- バックエンドは `artist` フィールドを返さない実装になっているが、本画面は曲名のみ表示するため問題は表面化しない（[API エンドポイント仕様書 §4.2](./APIエンドポイント仕様書.md) 参照）。

## 3. レイアウト

```
┌──────────────────────────────────────────────────────────────┐
│ Header                                                       │
├──────────────────────────────────────────────────────────────┤
│ ┃ アーティスト名                          全 N 件             │
│                                                              │
│ ┌──────────────────────────────────────────────────┐ ┌─────┐ │
│ │ [pagination]                                     │ │ ... │ │
│ │  1. 曲名                                         │ └─────┘ │
│ │  2. 曲名                                         │         │
│ │  ...                                             │         │
│ │ 15. 曲名                                         │         │
│ └──────────────────────────────────────────────────┘         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- `PageContainer` の中に `SectionTitle`（タイトル: `name`、件数: `result?.count`）→ `ListContainer` → `PaginatedList`。
- 各行は `NormalListItem` + `ListText`（曲名のみ）。
- 行クリックで `/song/{song.id}` へ遷移。

## 4. ページネーション

| 設定 | 値 |
| --- | --- |
| 1 ページの件数 | 15 |
| ベース URL | `/artist/` |
| クエリ | `name`（URL に組み込む） |
| URL ページ番号 | `page` パラメータ |

```ts
usePagination(result?.songs || [], {
    itemsPerPage: 15,
    baseUrl: "/artist/",
    currentUrlPage: page,
    query: name,
});
```

ページ URL は `buildPaginationUrl('/artist/', name, page)` で組み立てる:

| ページ番号 | URL |
| --- | --- |
| 1 | `/artist/{name}` |
| 2 以降 | `/artist/{name}/{page}` |

### 4.1 キーボード操作

`usePagination` が `keydown` リスナーを登録する。

| キー | 挙動 |
| --- | --- |
| `ArrowLeft` | 前のページへ（1 ページ目では無効） |
| `ArrowRight` | 次のページへ（最終ページでは無効） |

## 5. 状態と表示

| 状態 | 表示 |
| --- | --- |
| `loading === true` | `ListContainer` 内に `CircularProgress` |
| `error` が truthy | 「取得に失敗しました」 |
| `result.songs.length` が 0 | 「取得に失敗しました」 |
| 通常 | `PaginatedList`（ページネーション + 曲リスト） |

`SectionTitle` はローディング中も表示される（タイトル本文は `name`、件数表示は `result?.count` が `undefined` の間は出ない）。
