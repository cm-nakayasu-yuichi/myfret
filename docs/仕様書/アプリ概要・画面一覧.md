# アプリ概要・画面一覧

## 1. アプリ概要

myfret は ufret.jp のコード譜を独自 UI で再描画する Web アプリ。ufret.jp で配信されている楽曲のコード譜・ランキング・検索結果をスクレイピングで取得し、myfret 側で再構成して表示する。カポ・曲キーの変更による転調はフロント側で完結し、サーバへの再フェッチは行わない。

### 1.1 構成

- **フロントエンド**: Vite + React 19 + TypeScript + MUI (`http://localhost:5173`)
- **バックエンド**: Express + cheerio (axios) + Playwright (`http://localhost:3001`)
- 1 リポジトリに 2 プロジェクトを同居（ルート: フロント、`backend/`: バックエンド）
- フロントは `VITE_API_BASE_URL` (`.env`) でバックエンドの URL を解決

### 1.2 役割分担

| レイヤ | 主な責務 |
| --- | --- |
| フロントエンド | 画面描画、転調処理、コードポジション生成、コードクリック→ダイアグラム表示、テーマ管理、キャッシュ |
| バックエンド | ufret.jp のスクレイピング、レスポンスの正規化、CORS |

## 2. ルーティング一覧

すべてのルートは `src/app/AppRouter.tsx` で宣言し、`src/components/pages/<name>.tsx` のコンポーネントを直接渡す。

| パス | 画面名 | page ファイル | ヘッダ | URL パラメータ |
| --- | --- | --- | --- | --- |
| `/` | ホーム | `pages/home.tsx` | あり | なし |
| `/artist/:name` | アーティスト曲一覧 | `pages/artist.tsx` | あり | `name` |
| `/artist/:name/:page` | アーティスト曲一覧（ページ指定） | `pages/artist.tsx` | あり | `name`, `page` |
| `/song/:songId` | 曲（コード譜） | `pages/song.tsx` | あり | `songId` |
| `/search/songs/:keyword` | 曲検索結果 | `pages/search-songs.tsx` | あり | `keyword` |
| `/search/artists/:keyword` | アーティスト検索結果 | `pages/search-artists.tsx` | あり | `keyword` |
| `/search/artists/:keyword/:page` | アーティスト検索結果（ページ指定） | `pages/search-artists.tsx` | あり | `keyword`, `page` |
| `/ranking/songs` | 曲ランキング | `pages/ranking-songs.tsx` | あり | なし |
| `/ranking/songs/:page` | 曲ランキング（ページ指定） | `pages/ranking-songs.tsx` | あり | `page` |
| `/ranking/artists` | アーティストランキング | `pages/ranking-artists.tsx` | あり | なし |
| `/ranking/artists/:page` | アーティストランキング（ページ指定） | `pages/ranking-artists.tsx` | あり | `page` |
| `/codebook` | コードブック | `pages/codebook.tsx` | あり | なし |
| `/help` | ヘルプ | `pages/help.tsx` | あり | なし |
| `/test` | テスト（開発用） | `pages/test.tsx` | なし | なし |
| `*` | 404 | `pages/not-found.tsx` | なし | — |

### 2.1 ヘッダ表示の制御

ヘッダ有無は `src/utils/routeElement.tsx` の第 2 引数 `noHeader` で制御する。`true` を渡したルート (`/test` と `*`) のみヘッダ非表示。それ以外は `LayoutWithHeader` で包まれ、上部に `Header` が描画される。

```tsx
export const routeElement = (element: React.ReactNode, noHeader?: boolean) => {
    return noHeader ? element : <LayoutWithHeader>{element}</LayoutWithHeader>;
};
```

### 2.2 ページ番号付きルート

アーティスト曲一覧・検索結果（アーティスト）・ランキング系の 3 種類はページ番号付きルートを別途定義している。ページ番号が未指定の場合は同じ画面コンポーネントが「1 ページ目」として描画される。

## 3. 画面遷移

```
ホーム
├─ ヘッダ検索ボックス → 曲検索結果 / アーティスト検索結果
├─ ランキングセクションの項目 → 曲 / アーティスト
├─ 「もっと見る」リンク → 曲ランキング / アーティストランキング
└─ ヘッダメニュー → コードブック / ヘルプ / テーマ切替

曲検索結果 / アーティスト検索結果
└─ 検索結果項目 → 曲 / アーティスト

曲ランキング / アーティストランキング
└─ ランキング項目 → 曲 / アーティスト

アーティスト曲一覧
└─ 曲リスト → 曲

曲（コード譜）
├─ サイドバーのカポ・曲キー変更 → 同じ画面内で再転調
└─ コード文字列クリック → サイドバーにコードダイアグラム表示
```

ヘッダの検索・メニューは（ヘッダ非表示ルート以外の）すべての画面から利用可能。

## 4. ヘッダ

`src/components/layouts/header/Header.tsx`。`AppBar` 内に左からロゴ・検索ボックス・メニューボタンを配置する。

### 4.1 検索ボックス

`HeaderSearch.tsx`。テキスト入力 + 検索タイプ切替（「曲名」/「アーティスト名」）+ 検索ボタン。

| 検索タイプ | 遷移先 |
| --- | --- |
| 曲名（`song`） | `/search/songs/{encodeURIComponent(keyword)}` |
| アーティスト名（`artist`） | `/search/artists/{encodeURIComponent(keyword)}` |

- キーワードは前後の空白を trim し、空文字なら遷移しない。
- 検索タイプとプレースホルダ・遷移先 URL は `src/types/enums/SearchType.ts` で一元管理。

### 4.2 メニュー

`HeaderMenu.tsx`。右端のメニューアイコンをクリックするとドロワー（`Menu`）が開き、以下の項目を持つ。

1. **テーマ切替**: 「ライト」「ダーク」のトグルボタン
2. **コードブック**: `/codebook` へ遷移
3. **ヘルプ**: `/help` へ遷移

## 5. テーマ

`src/contexts/theme/ThemeContext.tsx`。

```ts
export type THEME = 'light' | 'dark';
```

- 状態は `ThemeProvider` が `useState` で保持し、`useThemeContext()` で参照する。
- `localStorage` キー `app.theme` に永続化。アプリ起動時に同キーから初期値を取得し、未設定時は `'light'`。
- 切替トリガはヘッダメニューの ToggleButtonGroup（4.2 参照）。`toggleTheme(newTheme)` が呼ばれると `useState` と `localStorage` を同時に更新する。

## 6. キャッシュ

詳細は [キャッシュ仕様書](./キャッシュ仕様書.md) を参照。概要のみ:

- `src/utils/classes/CacheManager.ts` が `localStorage` ベースの TTL 付きキャッシュを提供。
- キャッシュ対象は **ホーム情報・曲ランキング・アーティストランキング** のみ。TTL は 1 週間。
- 曲ページ・検索結果・アーティスト曲一覧は毎回フェッチ（キャッシュなし）。

## 7. データ取得

各画面が利用する hook と API エンドポイントの対応は以下。詳細は [API エンドポイント仕様書](./APIエンドポイント仕様書.md) を参照。

| 画面 | hook | API |
| --- | --- | --- |
| ホーム | `useGetHomeInfo` | `GET /api/home-info` |
| 曲 | `useGetSong` | `GET /api/song/:id` |
| アーティスト曲一覧 | `useGetSongsOfArtist` | `GET /api/artist/:name` |
| 曲検索結果 | `useSearchSongs` | `GET /api/search/songs/:keyword` |
| アーティスト検索結果 | `useSearchArtists` | `GET /api/search/artists/:keyword` |
| 曲ランキング | `useRankingSongs` | `GET /api/ranking/songs` |
| アーティストランキング | `useRankingArtists` | `GET /api/ranking/artists` |
