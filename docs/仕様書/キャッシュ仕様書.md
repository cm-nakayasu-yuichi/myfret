# キャッシュ仕様書

## 1. 概要

myfret のキャッシュは **フロントエンドの `localStorage` ベース** で実装されている。バックエンド側にキャッシュ層は無く、ufret.jp への再アクセスはバックエンドが毎回行う。

キャッシュの対象は **ホーム情報・曲ランキング・アーティストランキング** の 3 種類のみ。曲（コード譜）画面・検索結果画面・アーティスト曲一覧画面はキャッシュせず、毎回 API を叩く。

## 2. CacheManager

実装: `src/utils/classes/CacheManager.ts`。

### 2.1 型・定数

```ts
interface CacheItem<T> {
    data: T;
    timestamp: number;
}

export const CacheKey = {
    HOME_INFO: "home_info_cache",
    RANKING_SONGS: "ranking_songs_cache",
    RANKING_ARTISTS: "ranking_artists_cache",
} as const;

export const CacheDuration = {
    WEEK: 7 * 24 * 60 * 60 * 1000, // 1 週間
    DAY: 24 * 60 * 60 * 1000,      // 1 日
    HOUR: 60 * 60 * 1000,          // 1 時間
} as const;
```

- 現状コード上で実際に使われている TTL は `WEEK` のみ。`DAY` / `HOUR` は将来用の定義。
- 現状コード上で使われているキーは `CacheKey` の 3 つのみ。

### 2.2 CacheManager クラス

```ts
class CacheManager {
    constructor(duration: number);
    get<T>(key: string): T | null;
    set<T>(key: string, data: T): void;
    remove(key: string): void;
    clear(): void;
}
```

| メソッド | 挙動 |
| --- | --- |
| `get<T>(key)` | `localStorage` から `CacheItem<T>` を取り出す。`Date.now() - timestamp < duration` なら `data` を返す。期限切れなら **その場でキーを削除して `null`** を返す。未保存なら `null`。 |
| `set<T>(key, data)` | `{ data, timestamp: Date.now() }` を JSON 文字列化して `localStorage` に保存。 |
| `remove(key)` | `localStorage.removeItem(key)` のラッパ。 |
| `clear()` | `localStorage.clear()`（`localStorage` 全体を消す。プレフィックススコープでない点に注意）。 |

### 2.3 注意事項

- `localStorage` のキー名にプレフィックスは付かない（`home_info_cache` など素のキー名で保存される）。
- `clear()` は `localStorage` 全体を削除する。テーマ設定 (`app.theme`) も巻き添えになるため、現状コードからは呼ばれていない。
- `get()` で取り出した値の JSON パースに失敗した場合のフォールバックは無い。

## 3. キャッシュ利用方針

### 3.1 キャッシュを使う hook

| hook | キー | TTL | 備考 |
| --- | --- | --- | --- |
| `useGetHomeInfo` | `HOME_INFO` | `WEEK` | — |
| `useRankingSongs` | `RANKING_SONGS` | `WEEK` | フェッチ前に `HOME_INFO` を削除（3.3 参照） |
| `useRankingArtists` | `RANKING_ARTISTS` | `WEEK` | フェッチ前に `HOME_INFO` を削除（3.3 参照） |

3 つの hook はそれぞれモジュールスコープに `CacheManager` インスタンスを 1 つだけ生成して使い回す。

### 3.2 キャッシュを使わない hook

以下はキャッシュ無し（毎回 API を叩く）:

- `useGetSong`
- `useGetSongsOfArtsist`
- `useSearchSongs`
- `useSearchArtists`

### 3.3 ランキング遷移時のホーム情報キャッシュ削除

`useRankingSongs` と `useRankingArtists` は、**自身のキャッシュが無くて API を叩くタイミングで `CacheKey.HOME_INFO` を `remove()` する**。

```ts
// useRankingSongs.ts / useRankingArtists.ts より抜粋
cacheManager.remove(CacheKey.HOME_INFO);
setLoading(true);
const response = await rankingSongs();
cacheManager.set(CacheKey.RANKING_SONGS, response);
```

- 自身のキャッシュがヒットした場合は削除しない。
- 「ランキングが更新されたタイミングでホーム画面のランキング表示も更新する」ためのデータ整合性対策。ランキングを直接見にいくユーザーは、その後ホームに戻ったときに最新のランキングを見ることになる。

## 4. キャッシュのライフサイクル

```
[ユーザー初回アクセス（ホーム）]
  ↓
useGetHomeInfo → cache miss → API → HOME_INFO に set (TTL 1週間)

[1 週間以内に再度ホーム]
  ↓
useGetHomeInfo → cache hit → そのまま表示

[1 週間以内にランキング画面に遷移]
  ↓
useRankingSongs → cache miss → HOME_INFO を remove → API → RANKING_SONGS に set
  ↓
ホームに戻る → useGetHomeInfo → cache miss → API（最新を再取得）

[1 週間経過後]
  ↓
useGetHomeInfo → cache 期限切れ → CacheManager.get 内でキーを自動削除 → API → set
```

## 5. 仕様の境界

- ブラウザの `localStorage` を直接利用するため、**ブラウザ・端末をまたいでキャッシュは共有されない**。
- シークレットモードや `localStorage` をクリアした場合は、初回アクセスと同じ挙動。
- `localStorage` の容量制限（一般的に 5MB 程度）を超えた場合の挙動は未定義（`set` 内で例外が出るがハンドルしていない）。
