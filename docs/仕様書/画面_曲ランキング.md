# 画面仕様書: 曲ランキング

## 1. 概要

| 項目 | 値 |
| --- | --- |
| パス | `/ranking/songs/` / `/ranking/songs/:page` |
| 画面コンポーネント | `src/components/pages/ranking-songs.tsx` (`RankingSongsPage`) |
| ルート定義 | `src/app/routes/ranking-songs.tsx` |
| ヘッダ | あり |
| URL パラメータ | `page`（省略可） |
| 主目的 | ufret.jp の週間ランキング（曲）をページネーション付きで表示 |

ホーム画面の曲ランキングセクション「もっと見る」リンクからの遷移先。

## 2. データ取得

| 項目 | 値 |
| --- | --- |
| hook | `useRankingSongs` |
| API | `GET /api/ranking/songs/` |
| レスポンス型 | `SongListResponse` |
| キャッシュ | `CacheKey.RANKING_SONGS`、TTL 1 週間 |

```ts
interface SongListResponse {
    songs: SongListItem[];   // 各曲の id, name, artist, rank
    count: number;
}
```

### 2.1 キャッシュの副作用

`useRankingSongs` は自身のキャッシュが無くて API を叩くタイミングで `CacheKey.HOME_INFO` のキャッシュも削除する。詳細は [キャッシュ仕様書 §3.3](./キャッシュ仕様書.md) を参照。

## 3. レイアウト

```
┌──────────────────────────────────────────────────────────────┐
│ Header                                                       │
├──────────────────────────────────────────────────────────────┤
│ ┃ 楽曲ランキング                                              │
│                                                              │
│ ┌──────────────────────────────────────────────────┐         │
│ │ [pagination]                                     │         │
│ │ ① 曲名 - アーティスト                              │         │
│ │ ② 曲名 - アーティスト                              │         │
│ │ ③ 曲名 - アーティスト                              │         │
│ │  4. 曲名 - アーティスト                            │         │
│ │  ...                                              │         │
│ │ 20. 曲名 - アーティスト                            │         │
│ └──────────────────────────────────────────────────┘         │
└──────────────────────────────────────────────────────────────┘
```

- `PageContainer` の中に `SectionTitle`「楽曲ランキング」（**件数表示なし**）→ `ListContainer` → `PaginatedList`。
- 各行は `NormalListItem` + `RankingNumber` + `SongListText`。
- 行クリックで `/song/{song.id}` へ遷移。

> 補足: 件数 (`total`) が指定されていないため、`SectionTitle` は左の縦バー付きタイトルだけが表示される。

## 4. ページネーション

| 設定 | 値 |
| --- | --- |
| 1 ページの件数 | **20** |
| ベース URL | `/ranking/songs/` |
| クエリ | なし |
| URL ページ番号 | `page` パラメータ |

```ts
usePagination(result?.songs || [], {
    itemsPerPage: 20,
    baseUrl: "/ranking/songs/",
    currentUrlPage: page,
});
```

ページ URL は `buildPaginationUrl('/ranking/songs/', undefined, page)` で組み立てる:

| ページ番号 | URL |
| --- | --- |
| 1 | `/ranking/songs/` |
| 2 以降 | `/ranking/songs/{page}` |

### 4.1 キーボード操作

| キー | 挙動 |
| --- | --- |
| `ArrowLeft` | 前のページへ |
| `ArrowRight` | 次のページへ |

## 5. ランキング順位表示

各行の左に `RankingNumber` を配置する。順位は **API レスポンスの `song.rank` フィールド**（ループ内のページネーションインデックスではない）。

| 順位 | 見た目 |
| --- | --- |
| 1 位 | 金色アバター、太字 |
| 2 位 | 銀色アバター、太字 |
| 3 位 | 銅色アバター、太字 |
| 4 位以降 | 通常色アバター、通常ウェイト |

詳細は [共通 UI・レイアウト仕様書 §5](./共通UI・レイアウト仕様書.md#5-rankingnumber)。

## 6. 状態と表示

| 状態 | 表示 |
| --- | --- |
| `loading === true` | `CircularProgress` |
| `error` が truthy | 「取得に失敗しました」 |
| `result.songs.length` が 0 | 「取得に失敗しました」 |
| 通常 | `PaginatedList` |
