# 画面仕様書: アーティストランキング

## 1. 概要

| 項目 | 値 |
| --- | --- |
| パス | `/ranking/artists/` / `/ranking/artists/:page` |
| 画面コンポーネント | `src/components/pages/ranking-artists.tsx` (`RankingArtistsPage`) |
| ルート定義 | `src/app/routes/ranking-artists.tsx` |
| ヘッダ | あり |
| URL パラメータ | `page`（省略可） |
| 主目的 | ufret.jp のアーティストランキングをページネーション付きで表示 |

ホーム画面のアーティストランキングセクション「もっと見る」リンクからの遷移先。

## 2. データ取得

| 項目 | 値 |
| --- | --- |
| hook | `useRankingArtists` |
| API | `GET /api/ranking/artists/` |
| レスポンス型 | `ArtistListResponse` |
| キャッシュ | `CacheKey.RANKING_ARTISTS`、TTL 1 週間 |

```ts
interface ArtistListResponse {
    artists: ArtistListItem[];   // 各アーティストの id, name, rank
    count: number;
}
```

### 2.1 キャッシュの副作用

`useRankingArtists` は自身のキャッシュが無くて API を叩くタイミングで `CacheKey.HOME_INFO` のキャッシュも削除する。詳細は [キャッシュ仕様書 §3.3](./キャッシュ仕様書.md) を参照。

## 3. レイアウト

```
┌──────────────────────────────────────────────────────────────┐
│ Header                                                       │
├──────────────────────────────────────────────────────────────┤
│ ┃ アーティストランキング                  全 N 件             │
│                                                              │
│ ┌──────────────────────────────────────────────────┐         │
│ │ [pagination]                                     │         │
│ │ ① アーティスト名                                   │         │
│ │ ② アーティスト名                                   │         │
│ │ ③ アーティスト名                                   │         │
│ │  4. アーティスト名                                 │         │
│ │  ...                                              │         │
│ │ 20. アーティスト名                                 │         │
│ └──────────────────────────────────────────────────┘         │
└──────────────────────────────────────────────────────────────┘
```

- `PageContainer` の中に `SectionTitle`「アーティストランキング」（件数: `result?.count`）→ `ListContainer` → `PaginatedList`。
- 各行は `NormalListItem` + `RankingNumber` + `ArtistListText`。
- 行クリックで `/artist/{artist.id}` へ遷移。

> 補足: 曲ランキング画面は件数を表示しないが、本画面は件数を表示する。`SectionTitle` の `total` プロパティの渡し方の違い。

## 4. ページネーション

| 設定 | 値 |
| --- | --- |
| 1 ページの件数 | **20** |
| ベース URL | `/ranking/artists/` |
| クエリ | なし |
| URL ページ番号 | `page` パラメータ |

```ts
usePagination(result?.artists || [], {
    itemsPerPage: 20,
    baseUrl: "/ranking/artists/",
    currentUrlPage: page,
});
```

ページ URL は `buildPaginationUrl('/ranking/artists/', undefined, page)` で組み立てる:

| ページ番号 | URL |
| --- | --- |
| 1 | `/ranking/artists/` |
| 2 以降 | `/ranking/artists/{page}` |

### 4.1 キーボード操作

| キー | 挙動 |
| --- | --- |
| `ArrowLeft` | 前のページへ |
| `ArrowRight` | 次のページへ |

## 5. ランキング順位表示

各行の左に `RankingNumber` を配置する。順位は API レスポンスの `artist.rank` フィールド。色分けルールは [共通 UI・レイアウト仕様書 §5](./共通UI・レイアウト仕様書.md#5-rankingnumber) を参照（金・銀・銅・通常）。

## 6. 状態と表示

| 状態 | 表示 |
| --- | --- |
| `loading === true` | `CircularProgress` |
| `error` が truthy | 「取得に失敗しました」 |
| `result.artists.length` が 0 | 「取得に失敗しました」 |
| 通常 | `PaginatedList` |

## 7. ナビゲーション

| --- | --- |
| 行クリック | `/artist/{artist.id}` |

| `ArrowLeft` / `ArrowRight` | 同上 |
